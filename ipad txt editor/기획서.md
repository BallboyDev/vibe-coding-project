# ipad markdown editor 개발 명세서
> 프로젝트 명 : `아이패드 txt Editor`

## 1. 프로젝트 개요
- 본 프로젝트는 애플 아이패드에서 구동되는 텍스트 에디터 어플 개발 바이브 코딩 프로젝트이다.

### 목적
1. Gemini CLI를 활용한 바이브 코딩을 통해 아이패드 어플 개발 진행 프로젝트
2. 아이패드에서의 텍스트 기반 파일과 디렉토리에 대하여 생성, 수정, 삭제, 이름 변경 기능이 구현된 어플 개발
3. 주로 마크다운 파일에 대한 작업을 진행하기 위해 관련 기능 개발

### 개발 환경
- 작성된 내용 이외는 모두 Gemini CLI 에서 자율적으로 선정
1. AI Agent : Gemini CLI
2. 개발 툴 : Xcode

### 개발 프로세스
1. xcode에서 기초 프로젝트 생성
2. 향후 모든 코드 작성과 프로젝트 진행은 gemini cli에서 진행


## 2. 어플 설계
- 테마
    - 전체적으로 다크테마를 기반으로 개발을 진행한다.
    - 색상은 VSCode 또는 아이패드의 다크 테마와 유사한 색상을 기반으로 한다.
    - 자세한 데이터는 1차 적으로 Gemini CLI가 임의로 선택하고 변경사항이 있을시 추후 변경한다.

- 레이아웃
    - VScode와 같이 좌측에 트리구조의 네비게이션 영역을 배치한다.
        - 루트 디렉토리는 애플의 Documents (문서) 디렉토리로 시작한다.
    - 남은 영역에는 텍스트를 수정할 수 있는 에디터 영역을 배치한다.
    - 에디터 영역의 상단에는 현재 표시되고 있는 파일의 이름과 수정 여부, 저장 상태 등을 표시하는 파일 정보 영역을 배치한다.
        - vscode와 유사한 디자인을 사용하기 희망
    - 어플의 최 하단에는 어플을 사용하면서 도움이 될 유용한 정보를 표시할 수 있는 정보 영역을 배치한다.
        - 커서의 위치, 드래그 선택된 택스트 수, 파일 확장자, 파일 용량 등
        - 우선은 빠르게 기능 구현이 가능한 정보만 표시하고 추후 고도화를 통해 정보를 추가한다.

- 기능
    - 디렉토리와 파일의 생성, 삭제, 이름 변경 기능
        - 네비게이션 영역에서의 디렉토리 또는 파일를 보조클릭(긴 터치)시 컨텍스트 메뉴 표시 
        - 디렉토리 보조 클릭시 컨텍스트 메뉴에는 `디렉토리 생성`, `파일 생성`, `이름 변경`, `삭제` 표시
        - 파일 보조 클릭시 컨텍스트 메뉴에는 `이름 변경`, `삭제` 표시

    - 파일의 내용 수정 기능
        - 파일 선택 시 에디터 영역에 파일의 내용 표시
        - 파일 내용 수정 시 파일 저장을 하기 전까지는 원본 파일은 수정 하지 않는다.
        - 파일 내용 수정하고 저장하지 않았을 경우 파일의 보조 클릭으로 나오는 컨텍스트 메뉴에 `저장` 버튼 표시
        - 수정된 내용을 저장하기 위해서는 단축키 `cmd + s`를 누르거나 컨텍스트 메뉴의 `저장` 버튼 클릭을 누른 경우에만 원본 파일을 수정한다.

    - 마크다운 확장자에 대한 하이라이팅 기능
        - 파일의 확장자가 마크다운일 경우 기본적인 하이라이팅 기능 구현
        - 오픈소스 Swift 라이브러리를 활용한 마크다운 하이라이팅 기능 구현
        - 라이브러리는 우선 가장 인기 있고 안정적인 마크다운 라이브러리를 선택한다.
        
## 3. 프로젝트 구조
- 프로젝트 구조는 아래 규칙을 따른다.
    1. 디자인 관련 파일과 기능 구현 관련 파일을 구분한다.
    2. 메인 기능 파일 1개에 기능별 구현 파일들을 분리한다.
        - 기능 하나당 기능 파일 하나를 분리하고 동일 기능 사용시 코드를 중복 작성하지 않도록 한다.
    3. 모든 코드는 재사용성을 중요 하게 작성한다.


## 4. 장기적인 계획
